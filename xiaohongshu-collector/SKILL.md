# 社交媒体案例研究助手

帮助用户高效完成小红书/抖音等平台的案例收集与分析工作。

## 核心理念

**我不能做**: 直接访问小红书/抖音（需登录、有反爬）、自动截图

**我能做**:
- 研究行业背景，识别值得关注的品牌和活动
- 提供精确搜索词，让用户有针对性地截图
- 自动创建文件夹结构，用户只需拖放截图
- 分析用户提供的截图，生成报告

## 工作流程

```
Phase 1: 研究准备 (Claude完成)
    ↓
Phase 2: 创建文件夹 (Claude完成)
    ↓
Phase 3: 用户截图 (用户执行)
    ↓
Phase 4: 分析报告 (Claude完成)
```

---

## Phase 1: 研究准备

### 1.1 收集用户需求

向用户询问：

```
📋 社媒案例研究配置

请告诉我：

1. **研究主题/品牌**
   例如：瑞幸咖啡竞品、新茶饮营销、美妆品牌联名

2. **研究目的**
   例如：竞品分析、营销策略研究、内容灵感收集

3. **目标平台**
   a) 小红书
   b) 抖音
   c) 两者都要

4. **时间范围**（可选）
   例如：最近3个月、2024年全年

我将为您：
✓ 识别相关品牌/账号
✓ 找出值得关注的营销活动
✓ 生成精确搜索词清单
✓ 创建截图收集文件夹
```

### 1.2 执行行业研究

使用 WebSearch 工具搜索：
- `{品牌} 竞品 {年份} 市场`
- `{品牌} 小红书 营销 活动 案例`
- `{行业} 联名 campaign 案例分析`
- `{品牌} {竞品} 营销策略 对比`

### 1.3 整理研究成果

输出格式：

```markdown
## 竞品/品牌清单

| 品牌 | 定位 | 关注原因 |
|-----|------|---------|
| ... | ... | ... |

## 值得截图的活动/案例

### 高优先级 (必截)

#### 1. [活动名称]
- **品牌**: xxx
- **时间**: 2024年x月
- **搜索词**: `关键词1` / `关键词2`
- **截图要点**:
  - [ ] 官方发布海报
  - [ ] 产品图
  - [ ] 用户UGC (选2-3篇高赞)
- **为什么重要**: [一句话说明]

#### 2. [活动名称]
...

### 中优先级 (补充)
...
```

## Phase 2: 创建文件夹结构

### 2.1 自动创建文件夹

根据研究结果，使用 Bash 工具在用户桌面创建文件夹：

```bash
# 创建主文件夹和子文件夹
mkdir -p ~/Desktop/{项目名称}_研究_{日期}/{品牌1}/{活动1}
mkdir -p ~/Desktop/{项目名称}_研究_{日期}/{品牌1}/{活动2}
mkdir -p ~/Desktop/{项目名称}_研究_{日期}/{品牌2}/{活动1}
# ... 根据研究结果创建所有需要的文件夹
```

### 2.2 生成截图清单文件

在主文件夹中创建 `截图清单.md`：

```markdown
# 截图清单

## 使用说明
1. 打开小红书/抖音
2. 按下方搜索词搜索
3. 截图后直接拖入对应文件夹
4. 完成后告诉我"截图完成"

---

## 📁 {品牌1}/

### 📂 {活动1}/
搜索词: `xxx`
需要截图:
- [ ] 官方海报
- [ ] 产品图
- [ ] 用户笔记 x3

### 📂 {活动2}/
搜索词: `xxx`
需要截图:
- [ ] ...

---

## 📁 {品牌2}/
...
```

### 2.3 告知用户

```
✅ 文件夹已创建！

📁 位置: ~/Desktop/{项目名称}_研究_{日期}/

文件夹结构:
├── 品牌A/
│   ├── 活动1/  ← 把活动1的截图拖进来
│   └── 活动2/  ← 把活动2的截图拖进来
├── 品牌B/
│   └── ...
└── 截图清单.md  ← 打开这个看搜索词

📋 接下来：
1. 打开桌面上的「截图清单.md」
2. 按里面的搜索词去小红书/抖音搜索
3. 截图后拖入对应文件夹
4. 完成后回来告诉我"截图完成"
```

---

## Phase 3: 用户执行截图

用户按照截图清单手动完成截图，并将截图放入对应文件夹。

### 等待用户完成

当用户说"截图完成"、"done"、"完成了"等时，进入 Phase 4。

---

## Phase 4: 分析与报告

### 4.1 读取截图

使用 Read 工具读取用户截图（Claude可以看图）：

```
我来分析您收集的截图...
```

遍历文件夹中的图片，提取关键信息。

### 4.2 生成分析报告

在主文件夹中创建分析报告：

```markdown
# {项目名称} 案例分析报告

生成日期: {日期}

## Executive Summary
[一段话总结核心发现]

## 品牌/活动分析

### {品牌1}

#### {活动1}
- **活动概述**: ...
- **视觉风格**: ...
- **核心策略**: ...
- **效果数据**: (如果截图中可见)
- **可借鉴点**: ...

![截图](相对路径)

#### {活动2}
...

### {品牌2}
...

## 跨品牌对比

| 维度 | 品牌A | 品牌B | 品牌C |
|-----|------|------|------|
| 核心打法 | ... | ... | ... |
| 视觉风格 | ... | ... | ... |
| 目标人群 | ... | ... | ... |

## 关键洞察

1. **洞察1**: ...
2. **洞察2**: ...
3. **洞察3**: ...

## 建议

基于以上分析，建议...
```

### 4.3 可选：生成Slides大纲

如果用户需要做PPT：

```markdown
# Slides 大纲

## Slide 1: 封面
标题: {项目名称}

## Slide 2: 研究背景
- 研究目的
- 研究范围

## Slide 3-N: 案例分析
每个活动一页，包含：
- 活动名称
- [截图位置]
- 核心策略 (3个bullet points)

## Slide N+1: 对比总结
对比表格

## Slide N+2: 关键洞察
3-5个核心发现

## Slide N+3: 建议
可执行的下一步
```

## 示例对话

### 用户输入
"帮我研究瑞幸咖啡的竞品在小红书上的营销活动"

### Claude执行

1. **搜索行业信息** (WebSearch)
   - 瑞幸竞品清单
   - 各品牌小红书营销案例

2. **整理研究成果**
   - 竞品：库迪、Manner、幸运咖...
   - 重点活动：库迪×哪吒联名、Manner×LV...

3. **创建文件夹** (Bash)
   ```
   ~/Desktop/瑞幸竞品研究_2026-02-07/
   ├── 库迪/
   │   ├── 哪吒联名/
   │   └── 王一博代言/
   ├── Manner/
   │   └── LV联名/
   └── 截图清单.md
   ```

4. **等待用户截图**

5. **分析截图，生成报告**
